## 🌟 Interview Tips

1. **Identify Pattern First** (30 seconds)
   - Read problem once
   - Think about data structures
   - Match to familiar pattern

2. **State Your Approach** (1 minute)
   - "This looks like Sliding Window to me because..."
   - "I'll use this approach and time is O(n)..."

3. **Code the Template** (5-10 minutes)
   - Write template first
   - Modify for this problem
   - Less error-prone

4. **Walk Through Example** (2-3 minutes)
   - Trace through given example
   - Show understanding
   - Catches bugs early

5. **Discuss Complexity** (1 minute)
   - Time: how many operations?
   - Space: what's stored?
   - Can we do better?
